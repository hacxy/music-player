## 1. 产品概述

### 一句话北极星

> **让用户在本地浏览器中获得媲美 Apple Music 的沉浸式听歌体验，配合逐字高亮歌词，让每一句歌词都成为故事。**

### 产品定位

一个纯前端、零后端依赖的本地音乐播放器，专注于播放用户自行管理的音乐文件库。通过自动扫描静态资源目录，实现"放入即播放"的极简体验，同时提供专业级的逐字歌词同步和 Apple Music 风格的视觉呈现。

## 3. 用户故事

### Story 1: 首次使用 - 发现音乐

**As a** 用户,
**I want to** 打开应用后立即看到我的音乐库中所有歌曲,
**So that** 我可以快速找到想听的歌，无需复杂配置。

**Acceptance Criteria:**
- [ ] Given 用户打开应用, When 静态资源文件夹中已有音乐目录, Then 列表在 500ms 内加载完成
- [ ] Given 音乐目录结构正确（音频+歌词+封面）, When 扫描完成, Then 歌曲封面、歌名、时长正确显示
- [ ] Given 静态资源文件夹为空, When 打开应用, Then 显示友好的空状态引导

---

### Story 2: 播放音乐 - 核心体验

**As a** 用户,
**I want to** 点击歌曲后立即开始播放，并看到完整的播放界面,
**So that** 我能享受沉浸式的听歌体验。

**Acceptance Criteria:**
- [ ] Given 用户点击歌曲列表中的某首歌, When 点击完成, Then 音频在 200ms 内开始播放
- [ ] Given 歌曲正在播放, When 查看播放界面, Then 显示歌曲封面、歌名、歌手、进度条、歌词
- [ ] Given 歌曲播放中, When 用户点击暂停, Then 音频立即暂停，UI 状态同步更新
- [ ] Given 歌曲播放结束, When 无下一首, Then 停留在当前界面，显示重播按钮

---

### Story 3: 歌词同步 - 情感连接

**As a** 用户,
**I want to** 在听歌时看到歌词逐字高亮，跟随音乐节奏,
**So that** 我能更好地感受歌曲的情感，跟唱更轻松。

**Acceptance Criteria:**
- [ ] Given 歌曲有对应的歌词文件, When 播放到某句歌词, Then 当前歌词行高亮显示
- [ ] Given 歌词行正在播放, When 查看歌词区域, Then 当前字逐个高亮，动画流畅无卡顿
- [ ] Given 用户拖动进度条到新位置, When 释放进度条, Then 歌词立即跳转到对应位置，高亮状态同步更新
- [ ] Given 歌词区域较长, When 当前歌词超出可视区域, Then 自动滚动到当前歌词位置

---

### Story 4: 音量控制 - 个性化调节

**As a** 用户,
**I want to** 方便地调节音量大小,
**So that** 我能在不同环境下获得舒适的听歌体验。

**Acceptance Criteria:**
- [ ] Given 用户拖动音量滑块, When 拖动中, Then 音量实时变化
- [ ] Given 用户点击静音按钮, When 当前有音量, Then 音量归零，图标切换为静音状态
- [ ] Given 用户再次点击静音按钮, When 当前已静音, Then 恢复到之前的音量水平

---

### Story 5: 切换歌曲 - 流畅控制

**As a** 用户,
**I want to** 快速切换到上一首或下一首歌曲,
**So that** 我能连续欣赏音乐，无需手动选择。

**Acceptance Criteria:**
- [ ] Given 歌曲正在播放, When 用户点击下一首, Then 当前歌曲停止，下一首在 300ms 内开始播放
- [ ] Given 歌曲正在播放, When 用户点击上一首, Then 当前歌曲停止，上一首在 300ms 内开始播放
- [ ] Given 歌曲播放不到 3 秒, When 用户点击上一首, Then 跳转到上一首歌曲
- [ ] Given 歌曲播放超过 3 秒, When 用户点击上一首, Then 当前歌曲重新开始播放

## 4. 数据结构定义

### 4.1 音乐目录结构

```
static/
├── songs/
│   ├── song-001/                    # 每首歌一个目录
│   │   ├── audio.mp3                # 音频文件（支持 mp3/wav/ogg/flac）
│   │   ├── cover.jpg                # 封面图片（支持 jpg/png/webp）
│   │   └── lyrics.json              # 歌词文件（JSON 格式）
│   │
│   ├── song-002/
│   │   ├── audio.mp3
│   │   ├── cover.png
│   │   └── lyrics.json
│   │
│   └── ...
```

**目录命名规范：**
- 目录名使用 kebab-case（如 `love-story`, `bohemian-rhapsody`）
- 目录名将作为歌曲的默认显示名称（可被歌词文件中的元数据覆盖）

### 4.2 歌词文件格式 (lyrics.json)

```json
{
  "audio_file": "audio.mp3",
  "duration_seconds": 245.5,
  "metadata": {
    "title": "歌曲名称",
    "artist": "歌手名称",
    "album": "专辑名称"
  },
  "lrc_text": "[00:00.00]歌曲名称\n[00:05.20]第一句歌词\n[00:10.40]第二句歌词",
  "token_timestamps": [
    {
      "line_index": 0,
      "tokens": [
        { "text": "第", "start": 5.20, "end": 5.35 },
        { "text": "一", "start": 5.35, "end": 5.50 },
        { "text": "句", "start": 5.50, "end": 5.65 },
        { "text": "歌", "start": 5.65, "end": 5.80 },
        { "text": "词", "start": 5.80, "end": 5.95 }
      ]
    },
    {
      "line_index": 1,
      "tokens": [
        { "text": "第", "start": 10.40, "end": 10.55 },
        { "text": "二", "start": 10.55, "end": 10.70 },
        { "text": "句", "start": 10.70, "end": 10.85 },
        { "text": "歌", "start": 10.85, "end": 11.00 },
        { "text": "词", "start": 11.00, "end": 11.15 }
      ]
    }
  ]
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `audio_file` | string | 是 | 音频文件名，相对于当前目录 |
| `duration_seconds` | number | 是 | 音频总时长（秒） |
| `metadata.title` | string | 否 | 歌曲标题，覆盖目录名 |
| `metadata.artist` | string | 否 | 歌手名称 |
| `metadata.album` | string | 否 | 专辑名称 |
| `lrc_text` | string | 是 | LRC 格式歌词，用于回退显示 |
| `token_timestamps` | array | 是 | 逐字时间戳数组 |
| `token_timestamps[].line_index` | number | 是 | 对应 LRC 歌词的行索引（从 0 开始） |
| `token_timestamps[].tokens` | array | 是 | 该行的所有字/词 |
| `token_timestamps[].tokens[].text` | string | 是 | 字符内容 |
| `token_timestamps[].tokens[].start` | number | 是 | 开始时间（秒，相对于音频开头） |
| `token_timestamps[].tokens[].end` | number | 是 | 结束时间（秒） |

### 4.3 扫描结果数据结构（应用内部使用）

```typescript
interface Song {
  id: string;                    // 唯一标识，基于目录路径生成
  directoryName: string;         // 目录名
  title: string;                 // 歌曲标题（来自 metadata 或目录名）
  artist: string;                // 歌手（来自 metadata，默认 "Unknown Artist"）
  album: string;                 // 专辑（来自 metadata，默认 "Unknown Album"）
  duration: number;              // 时长（秒）
  audioSrc: string;              // 音频文件 URL
  coverSrc: string;              // 封面图片 URL
  lyrics: LyricsData | null;     // 歌词数据，无歌词文件时为 null
}

interface LyricsData {
  lines: LyricsLine[];
  rawLrc: string;                // 原始 LRC 文本
}

interface LyricsLine {
  index: number;
  startTime: number;             // 该行开始时间（秒）
  endTime: number;               // 该行结束时间（秒）
  text: string;                  // 完整文本
  tokens: LyricsToken[];         // 逐字数据
}

interface LyricsToken {
  text: string;
  start: number;
  end: number;
}
```

## 5. UI/UX 要求

### 5.1 设计原则

1. **沉浸优先**：歌词和封面是视觉焦点，控制元素不喧宾夺主
2. **动效克制**：动画服务于功能，不做纯装饰性动效
3. **深色主题**：减少视觉干扰，突出内容，符合音乐播放器的使用场景（多为夜间/低光环境）
4. **触摸友好**：所有可交互元素最小点击区域 44x44px

#### 逐字高亮实现

- **当前字**：白色 (#FFFFFF)，完整不透明度
- **已播放字**：白色，50% 不透明度
- **未播放字**：白色，30% 不透明度
- **动画效果**：颜色渐变过渡，无跳跃感

#### 自动滚动行为

- 当前歌词行始终居中显示
- 滚动使用 `scroll-behavior: smooth`
- 用户手动滚动后，暂停自动滚动 5 秒
- 点击歌词行可跳转到对应时间点

## 6. 技术约束

### 6.1 技术栈

| 类别 | 选择 | 版本 | 理由 |
|------|------|------|------|
| 框架 | React | 18.x | 组件化开发，生态成熟 |
| 构建工具 | Vite | 5.x | 开发热更新快，构建产物小 |
| 样式方案 | Tailwind CSS | 4.x | 原子化 CSS，开发效率高 |
| 状态管理 | Zustand | 4.x | 轻量，适合中等复杂度状态 |
| 音频处理 | Web Audio API + HTMLAudioElement | - | 浏览器原生，无额外依赖 |

### 6.2 性能要求

| 指标 | 目标 | 测量方式 |
|------|------|----------|
| 首次加载 (FCP) | < 1.5s | Lighthouse |
| 歌曲列表渲染 (100 首) | < 500ms | Performance API |
| 播放响应延迟 | < 100ms | 从点击到音频播放 |
| 歌词同步精度 | < 50ms | 人工对比测试 |
| 内存占用 | < 200MB | Chrome DevTools |
| 动画帧率 | > 60fps | Chrome DevTools |

### 6.3 浏览器兼容性

| 浏览器 | 最低版本 | 优先级 |
|--------|----------|--------|
| Chrome | 90+ | P0 |
| Firefox | 90+ | P0 |
| Safari | 15+ | P0 |
| Edge | 90+ | P1 |

### 6.4 静态资源限制

- 单个音频文件：建议 < 50MB（避免加载过慢）
- 单个封面图片：建议 < 2MB
- 歌词文件：通常 < 50KB
- 总歌曲数量：设计支持 1000 首，MVP 验证 100 首

### 6.5 关键技术风险

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 歌词同步精度不足 | 中 | 高 | 使用 requestAnimationFrame 高频检测，而非 setInterval |
| 大量歌曲扫描性能 | 中 | 中 | 实现虚拟滚动列表，延迟加载封面 |
| 音频格式兼容性 | 低 | 高 | 提供格式转换指南，MVP 仅保证 mp3 支持 |
| 毛玻璃效果性能 | 低 | 中 | 提供降级方案，低端设备关闭模糊效果 |

## 7. MVP 范围界定

### 7.1 MVP 包含（必须实现）

| 功能 | 范围说明 |
|------|----------|
| 静态资源扫描 | 支持扫描 `static/songs/` 目录下的所有子目录 |
| 音频播放 | 支持 mp3 格式，播放/暂停/进度拖拽/音量控制 |
| 歌曲列表 | 基础列表展示，点击播放，当前播放高亮 |
| 逐字歌词 | 根据 token_timestamps 实时高亮，自动滚动 |

| 封面显示 | 歌曲封面大图展示，列表缩略图 |
| 底部播放栏 | 固定在底部的迷你播放控制栏 |

### 7.2 MVP 不包含（明确不做）

| 功能 | 不做的理由 | 何时考虑 |
|------|------------|----------|
| 用户认证/登录 | 纯前端项目，无需用户系统 | 如需云同步功能时 |
| 后端服务 | 需求明确为纯前端 | 如需多设备同步时 |
| 播放队列管理 | MVP 专注单曲播放体验 | v1.1 |
| 播放模式（循环/随机） | MVP 先验证核心体验 | v1.1 |
| 歌曲搜索/过滤 | 歌曲量小时不需要 | v1.2 |
| 歌曲收藏/评分 | 增加复杂度，MVP 后验证 | v1.2 |
| 音频可视化 | 非核心功能 | v2.0 |
| 响应式移动端 | 先打磨桌面体验 | v2.0 |
| 键盘快捷键 | 锦上添花 | v1.1 |
| 歌词编辑 | 需求不明确 | 用户反馈后决定 |
| 多语言支持 | MVP 仅支持中文 | 国际化需求时 |
| 离线缓存 (Service Worker) | 增加复杂度 | 网络不稳定场景验证后 |
| 音频均衡器 | 高级功能 | v2.0 |
| 歌词导入/导出 | 非核心 | 用户反馈后 |

### 7.3 MVP 验证目标

| 目标 | 指标 | 达标标准 |
|------|------|----------|
| 核心可用性 | 能否完成完整听歌流程 | 从打开到听完一首歌无报错 |
| 歌词体验 | 逐字高亮是否流畅 | 人工测试同步误差可接受 |
| 视觉体验 | 是否有 Apple Music 质感 | 设计评审通过 |
| 性能基础 | 列表和播放是否流畅 | 100 首歌场景下无明显卡顿 |
